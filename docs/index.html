Introduction

Weirdstore is a store management system designed to simplify and enhance the way you run your store. Whether you are managing a small boutique or a large retail chain, Weirdstore provides the tools you need to succeed.

Services Overview

Category Service

Handles all operations related to categories.

  • Manage category data: Create, Read, Update, Delete.
  • Endpoints:
    • POST /: Create Category
    • GET /categories: List Categories
    • GET /:id: Get Category By Id
    • PUT /:categoryId: Update Category
    • DELETE /:categoryId: Delete Category By ID

Order Service

Manages customer orders and order processing.

  • Handles order creation, retrieval, and updates.
  • Endpoints:
    • POST /: Create An Order
    • GET /mine: Get User Orders
    • PUT /:id/pay: Mark Order As Paid
    • GET /total-sales: Calculate Total Sales

Product Service

Manages product data, including adding, updating, and fetching products.

  • Manages stock and inventory.
  • Endpoints:
    • POST /: Add Product
    • GET /top: Fetch Top Products
    • POST /:id/reviews: Add Product Review

Upload Service

Handles file uploads, such as product images and other media files.

User Service

Manages user authentication and user data.

  • Endpoints:
    • POST /auth: Login
    • GET /profile: Get User Profile
    • PUT /profile: Update User Profile

Architecture

Microservices: All services are independently developed and deployed, ensuring scalability and flexibility.

Database: MongoDB for consistent and scalable storage.

Communication: RESTful APIs and JWT-based authentication ensure secure communication between services.

Logging: Each service has built-in logging to help monitor and debug the system.

Authentication: JWT is used for secure communication between services, ensuring that only authorized users can access protected endpoints.

Setup Instructions

  1. Install dependencies for each service: 
    npm install
  2. Create a `.env` file for each service with the following content: 
    MONGO_URI=
JWT_SECRET=
LOGGER_PATH=
PORT=
  3. Run each service: 
    npm start
  4. Test with tools like Postman to ensure endpoints are functional.
